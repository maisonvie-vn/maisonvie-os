# TASK-0035 — Menu Item Costing Foundation

## Objective
The objective of TASK-0035 is to implement the **Menu Item Costing Foundation** in MVOS Studio. This enables Maison Vie to define mappings between sellable menu items and one or more recipe components (e.g. main dish, sauce, garnish) and calculate theoretical menu item costs based on selected recipe costing runs, flagging incomplete data in a detailed breakdown.

## Data Structure

### Menu Item Recipe Mapping Fields
* `id` (UUID): Unique mapping identifier.
* `menuItemId` (UUID): Reference to menu item.
* `recipeId` (UUID): Reference to Recipe.
* `componentType` (VARCHAR): Component enum (`main`, `sauce`, `garnish`, `side`, `prep`, `bread`, `petit_four`, `amuse_bouche`, `other`).
* `costBasis` (VARCHAR): Cost basis enum (`per_portion`, `full_recipe`).
* `quantityMultiplier` (NUMERIC): Quantity factor.
* `isActive` (BOOLEAN): Status toggle.
* `notes` (TEXT).

### Menu Item Costing Run Fields
* `id` (UUID): Unique run identifier.
* `runNumber` (VARCHAR): Unique run number (e.g. MIC-20260630-01).
* `runDate` (DATE).
* `sourceRecipeCostingRunId` (UUID, Nullable): Reference to Recipe run.
* `status` (VARCHAR): Status enum (`draft`, `completed`, `completed_with_warnings`, `failed`, `cancelled`).
* `notes` (TEXT).
* `createdBy` (VARCHAR).
* `startedAt` (TIMESTAMPTZ).
* `completedAt` (TIMESTAMPTZ).

## Out of Scope & Limitations
* **Menu Pricing & Profitability**: No menu pricing, recommended selling prices, gross profit, food cost percentage, or actual period food cost calculations.
* **POS & Sales Integration**: No actual POS consumption logic or sales inventory deductions.
* **Multi-Branch Readiness**: No branch ownership, multi-branch filters, or branch permissions. RFC-0010 remains Proposed.
* **Supabase Integration**: Stores changes to `localStorage` for mocking capabilities.
