# TASK-0034 — Recipe Costing Foundation

## Objective
The objective of TASK-0034 is to implement the **Recipe Costing Foundation** in MVOS Studio. This enables Maison Vie to calculate the theoretical recipe cost based on recipe ingredients, unit conversions, and the latest Weighted Average Cost (WAC) per ingredient, flagging incomplete data in a detailed breakdown list.

## Data Structure

### Recipe Costing Run Fields
* `id` (UUID): Unique run identifier.
* `runNumber` (VARCHAR): Unique run number (e.g. RCC-20260630-01).
* `runDate` (DATE).
* `sourceWacRunId` (UUID, Nullable): Reference to WAC run.
* `status` (VARCHAR): Status enum (`draft`, `completed`, `completed_with_warnings`, `failed`, `cancelled`).
* `notes` (TEXT).
* `createdBy` (VARCHAR).
* `startedAt` (TIMESTAMPTZ).
* `completedAt` (TIMESTAMPTZ).
* `createdAt` (TIMESTAMPTZ).
* `updatedAt` (TIMESTAMPTZ).

### Recipe Costing Item Fields
* `id` (UUID): Unique costing record identifier per recipe.
* `recipeCostingRunId` (UUID): Reference to run.
* `recipeId` (UUID): Reference to Recipe.
* `totalRecipeCost` (NUMERIC, Nullable).
* `costPerPortion` (NUMERIC, Nullable): Total cost divided by portion/yield count.
* `portionCount` (NUMERIC, Nullable).
* `costingStatus` (VARCHAR): Status enum (`pending`, `calculated`, `calculated_with_warnings`, `no_ingredients`, `incomplete`, `error`).
* `warningMessage` (TEXT, Nullable).
* `calculatedLineCount` (INT).
* `warningLineCount` (INT).
* `totalLineCount` (INT).

### Recipe Costing Line Fields
* `id` (UUID): Unique line costing identifier.
* `recipeCostingItemId` (UUID): Reference to recipe item costing.
* `recipeIngredientId` (UUID, Nullable): Reference to recipe ingredient line.
* `recipeId` (UUID).
* `ingredientId` (UUID, Nullable): Reference to Ingredient Master.
* `ingredientName` (VARCHAR).
* `recipeQuantity` (NUMERIC).
* `recipeUnit` (VARCHAR).
* `costingUnit` (VARCHAR).
* `normalizedQuantity` (NUMERIC).
* `wacUnitCost` (NUMERIC).
* `lineCost` (NUMERIC).
* `lineStatus` (VARCHAR): Status enum (`pending`, `calculated`, `missing_ingredient`, `missing_quantity`, `missing_unit`, `missing_wac`, `missing_conversion`, `incomplete`, `error`).
* `warningMessage` (TEXT, Nullable).

## Out of Scope & Limitations
* **Menu Pricing & Profitability**: No menu pricing screens, margin reports, actual COGS calculations, or sales profit analyses.
* **Direct Cost Mutation**: Theoretical costs are derived strictly from recipe, conversion, and WAC configurations.
* **Warehouse / Locations**: Single stock container assumption.
* **Multi-Branch Readiness**: No branch ownership, multi-branch filters, or branch permissions. RFC-0010 remains Proposed.
* **Supabase Integration**: Stores changes to `localStorage` for mocking capabilities.
