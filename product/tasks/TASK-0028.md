# TASK-0028 — Purchase Order Foundation

## Objective
The objective of TASK-0028 is to implement the **Purchase Order Foundation** in MVOS Studio. This enables Maison Vie to create structured purchase orders and order lines directed to suppliers, linking back to internal purchase requests, and preparing for future inventory receiving and billing controls.

## Data Structure

### Purchase Order Header Fields
* `id` (UUID): Unique order identifier.
* `purchaseOrderCode` (VARCHAR, Nullable): Purchase order code (e.g., PO-20260628-01).
* `purchaseRequestId` (UUID, Nullable): Reference to internal purchase request.
* `supplierId` (UUID): Reference to supplier.
* `orderDate` (DATE): Date order created.
* `expectedDeliveryDate` (DATE, Nullable): Expected delivery date.
* `orderedBy` (VARCHAR): Creator of order.
* `department` (VARCHAR): Department enum (`boh`, `foh`, `bar`, `reservation`, `management`, `finance`, `inventory`, `stewarding`, `housekeeping`, `maintenance`, `other`).
* `priority` (VARCHAR): Priority enum (`low`, `medium`, `high`, `urgent`).
* `status` (VARCHAR): Status enum (`draft`, `sent`, `supplier_confirmed`, `partially_received`, `received`, `cancelled`, `closed`, `archived`).
* `deliveryAddressNote` (TEXT, Nullable): Delivery address.
* `deliveryTimeNote` (TEXT, Nullable): Delivery window.
* `qualityNote` (TEXT, Nullable): Quality checks required.
* `internalNote` (TEXT, Nullable).
* `sentAt` (TIMESTAMPTZ, Nullable).
* `confirmedAt` (TIMESTAMPTZ, Nullable).
* `closedAt` (TIMESTAMPTZ, Nullable).
* `createdAt` (TIMESTAMPTZ).
* `updatedAt` (TIMESTAMPTZ).

### Purchase Order Line Fields
* `id` (UUID): Unique line identifier.
* `purchaseOrderId` (UUID): Reference to parent order.
* `purchaseRequestLineId` (UUID, Nullable): Reference to source request line.
* `ingredientMasterId` (UUID, Nullable): Optional reference to Ingredient Master.
* `lineOrder` (INT): Sorting order.
* `itemName` (TEXT, Nullable): Item name description if no master ingredient is linked.
* `orderedQuantity` (NUMERIC, Nullable).
* `orderedUnit` (VARCHAR, Nullable).
* `supplierItemNote` (TEXT, Nullable).
* `qualityRequirementNote` (TEXT, Nullable).
* `deliveryNote` (TEXT, Nullable).
* `status` (VARCHAR): Status enum (`open`, `ordered`, `supplier_confirmed`, `partially_received`, `received`, `cancelled`, `closed`).
* `createdAt` (TIMESTAMPTZ).
* `updatedAt` (TIMESTAMPTZ).

## Out of Scope & Limitations
* **Supplier Pricing**: No supplier price lists, unit prices, line totals, order totals, price history, or contract pricing.
* **Costing & WAC**: No costing calculations, food cost, WAC, or recipe rollup.
* **VAT & Billing**: No VAT calculation, invoice management, debt tracking, or payment tracking.
* **Inventory Receiving**: No inventory receiving logs or stock movements. Statuses `partially_received` and `received` are manual enums only.
* **Supplier Selection**: No automatic supplier selection or supplier scoring.
* **AI & Optimization**: No AI purchase recommendations, forecasting, or production planning.
* **Email System**: No automatic emails to suppliers.
* **Multi-Branch Readiness**: No branch ownership, multi-branch filters, or branch permissions. RFC-0010 remains Proposed.
* **Supabase Integration**: Stores changes to `localStorage` for mocking capabilities.
