# TASK-0022 — Menu Catalog Foundation

## Objective
The objective of TASK-0022 is to build the **Menu Catalog Foundation** inside MVOS Studio. This enables structured tracking and cataloging of restaurant menus (À la carte, business lunch, degustation, travel agency set menus, tour group menus, event menus) and their respective menu items (dishes and beverage items) including multi-lingual names, display orders, allergy notes, dietary notes, pregnancy notes, BOH preparation directives, and FOH service directives.

## Data Structure

### Menu Catalog Fields
* `id` (UUID): Unique menu identifier.
* `menuName` (VARCHAR): Name of the menu.
* `menuCode` (VARCHAR, Nullable): Code of the menu catalog.
* `menuType` (VARCHAR): Menu type enums (`a_la_carte`, `business_lunch`, `degustation`, `travel_agency`, `tour_group`, `private_dining`, `event`, `beverage`, `wine`, `seasonal`, `special`, `other`).
* `status` (VARCHAR): Menu status (`draft`, `active`, `paused`, `archived`).
* `description` (TEXT, Nullable).
* `targetGuestType` (VARCHAR, Nullable): Guest type enums (`walk_in`, `a_la_carte_guest`, `tour_group`, `travel_agency`, `private_dining`, `corporate`, `event_guest`, `vip`, `internal`, `other`).
* `effectiveFrom` (DATE, Nullable).
* `effectiveTo` (DATE, Nullable).
* `languageNote` (TEXT, Nullable).
* `internalNote` (TEXT, Nullable).
* `ownerName` (VARCHAR, Nullable).

### Menu Item Fields
* `id` (UUID): Unique item identifier.
* `menuCatalogId` (UUID): References the parent menu catalog.
* `itemCode` (VARCHAR, Nullable): Code of the item.
* `itemNameFr` (VARCHAR, Nullable): French name.
* `itemNameEn` (VARCHAR, Nullable): English name.
* `itemNameVi` (VARCHAR, Nullable): Vietnamese name.
* `itemNameJa` (VARCHAR, Nullable): Japanese name.
* `itemDescriptionVi` (TEXT, Nullable).
* `itemDescriptionEn` (TEXT, Nullable).
* `section` (VARCHAR): Section enums (`amuse_bouche`, `starter`, `soup`, `salad`, `fish`, `meat`, `poultry`, `vegetarian`, `cheese`, `dessert`, `beverage`, `wine`, `other`).
* `courseType` (VARCHAR, Nullable): Course enums (`single_item`, `course_1`, `course_2`, `course_3`, `course_4`, `course_5`, `course_6`, `optional_choice`, supplement`, `other`).
* `itemType` (VARCHAR): Item enums (`food`, `beverage`, `wine`, `cocktail`, `mocktail`, `coffee_tea`, `other`).
* `status` (VARCHAR): Item status (`draft`, `active`, `paused`, `archived`).
* `allergyNote` (TEXT, Nullable).
* `dietaryNote` (TEXT, Nullable).
* `pregnancyNote` (TEXT, Nullable).
* `serviceNote` (TEXT, Nullable).
* `kitchenNote` (TEXT, Nullable).
* `displayOrder` (INTEGER).

## Out of Scope & Limitations
* **Recipe Costing & Ingredients**: No ingredient list tracking, yield calculation, raw ingredient costing, or WAC inventory evaluation.
* **Pricing Engine**: No price fields or price calculation (margin, VAT, markup).
* **POS Sync**: No connection to POS systems.
* **Supabase Integration**: Currently stores changes to `localStorage` for mocking capabilities. Database triggers and real client queries will be wired in future development stages.
