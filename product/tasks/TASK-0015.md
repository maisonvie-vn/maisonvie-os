# TASK-0015 — Daily Operations Checklist Foundation

## Objective
The objective of TASK-0015 is to build the **Daily Operations Checklist Foundation** inside MVOS Studio. This enables creating operational checklist templates, generating daily dated checklist runs for departments/shifts, and checking items to enforce operational discipline.

## Data Structure

### Checklist Template Fields
* `id` (UUID): Unique template identifier.
* `title` (VARCHAR): Name of checklist layout.
* `description` (TEXT, Nullable).
* `department` (VARCHAR): Target department (`foh`, `boh`, `reservation`, `email`, etc.).
* `shiftType` (VARCHAR): Shift enums (`opening`, `pre_service`, `lunch`, `dinner`, `closing`, `full_day`, `event`, `other`).
* `status` (VARCHAR): Template status (`draft`, `active`, `archived`).
* `ownerName` (VARCHAR): Assigned template manager.
* `relatedSopDocumentId` (UUID, Nullable): Foreign key to the associated `sop_documents` record.
* `items` (JSONB): Array of template items (`id`, `label`, `required`, `order`).

### Checklist Run Fields
* `id` (UUID): Unique run identifier.
* `templateId` (UUID): Foreign key referencing the template layout.
* `runDate` (DATE): Target date.
* `department` / `shiftType` (VARCHAR).
* `status` (VARCHAR): Lifecycle status (`open`, `in_progress`, `completed`, `overdue`, `cancelled`).
* `responsibleName` (VARCHAR): Staff name executing the check.
* `completedItems` (JSONB): Array of run items (`itemId`, `checked`, `note`, `checkedAt`).
* `note` (TEXT, Nullable): General run comments.
* `completedAt` (TIMESTAMP, Nullable).

## Features Implemented

### 1. Summary Cards
Shows real statistics calculated from the runs list:
* **Checklist hôm nay**: Run count scheduled for today.
* **Đang thực hiện**: Runs with status `in_progress`.
* **Đã hoàn thành**: Completed runs with status `completed`.
* **Quá hạn**: Unfinished runs past their scheduled date.
* **Mẫu đang áp dụng**: Total active templates available.

### 2. Checklist Template Configurator
Allows configuring dynamic check items (adding/removing inputs, setting required flag) and linking templates to SOP documents.

### 3. Checklist Runs Execution Board
Allows instantiating runs, checking/unchecking items, writing notes, and updating progress statuses (`Lưu tiến độ`, `Đánh dấu đã hoàn thành` - checking all required items is validated, `Đánh dấu quá hạn`, and `Hủy checklist`).

## Out of Scope & Limitations
* **Full Audit Audit Trail**: No detailed change tracking logs for individual clicks.
* **AI checklist monitoring**: No automated checks or camera/sensor monitoring integrations.
* **Supabase Integration**: Currently stores changes to `localStorage` for mocking capabilities. Database triggers and real client queries will be wired in future development stages.
