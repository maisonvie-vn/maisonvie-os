# TASK-0013 — SOP Library Foundation

## Objective
The objective of TASK-0013 is to build the **SOP Library Foundation** inside MVOS Studio. This stores, catalogs, and version-controls basic Standard Operating Procedures (SOPs) based on operational improvements, establishing a centralized knowledge center for Maison Vie.

## Data Structure

### SOP Document Fields
* `id` (UUID): Unique SOP identifier.
* `title` (VARCHAR): Title of the procedure.
* `summary` (TEXT, Nullable): High-level scope and context.
* `content` (TEXT): Step-by-step operating guidelines.
* `category` (VARCHAR): Category enums (`reservation`, `email`, `foh_service`, `boh_kitchen`, etc.).
* `status` (VARCHAR): Lifecycle status (`draft`, `reviewing`, `active`, `archived`).
* `version` (INT): Document revision count (defaults to 1).
* `ownerName` (VARCHAR): Assigned process owner.
* `relatedImprovementActionId` (UUID, Nullable): Foreign key to the associated `improvement_actions` record.
* `effectiveDate` (DATE, Nullable): Starting date of compliance.
* `createdAt` / `updatedAt` (TIMESTAMP).

## Features Implemented

### 1. Summary Cards
Shows real statistics calculated from the SOPs list:
* **Tổng SOP**: Total SOPs logged.
* **Đang áp dụng**: SOPs with status `active`.
* **Bản nháp**: SOPs with status `draft`.
* **Đang xem xét**: SOPs with status `reviewing`.
* **Lưu trữ**: SOPs with status `archived`.

### 2. SOP Registration Form
Allows logging new SOP documents with references to resolved improvement actions. Includes input validations in Vietnamese.

### 3. Detailed View & Action Panel
Displays summaries, step-by-step text, version logs, and related actions, and allows updating document status (`Chuyển sang đang xem xét`, `Đánh dấu đang áp dụng`, and `Lưu trữ SOP`).

---

## Out of Scope & Limitations
* **AI SOP Generation**: No automated generation of procedures using LLMs.
* **Complex Approval Chain**: No hierarchical manager sign-offs or digital signatures.
* **Supabase Integration**: Currently stores changes to `localStorage` for mocking capabilities. Database triggers and real client queries will be wired in future development stages.
