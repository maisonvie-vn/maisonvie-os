# TASK-0030 — Inventory Movement Foundation

## Objective
The objective of TASK-0030 is to implement the **Inventory Movement Foundation** in MVOS Studio. This enables Maison Vie to record structured stock movement events (such as purchase receipt, manual adjustments, waste, spoilage, staff meals, and internal usage) without doing valuation, costing, or WAC.

## Data Structure

### Inventory Movement Fields
* `id` (UUID): Unique movement identifier.
* `movementNumber` (VARCHAR): Unique movement sequence code (e.g., IM-20260630-01).
* `ingredientMasterId` (UUID): Reference to Ingredient Master.
* `movementDate` (DATE): Date movement was recorded.
* `movementType` (VARCHAR): Type enum (`purchase_receipt`, `manual_adjustment`, `waste`, `spoilage`, `staff_meal`, `internal_use`, `correction`).
* `direction` (VARCHAR): Direction enum (`in`, `out`, `adjustment`).
* `quantity` (NUMERIC): Positive quantity moved.
* `unit` (VARCHAR, Nullable).
* `sourceType` (VARCHAR, Nullable): Source system identifier (e.g., `goods_receipt`).
* `sourceId` (UUID, Nullable): Reference to source header.
* `sourceLineId` (UUID, Nullable): Reference to source line item.
* `referenceNumber` (VARCHAR, Nullable).
* `reason` (TEXT, Nullable).
* `notes` (TEXT, Nullable).
* `createdBy` (VARCHAR, Nullable).
* `createdAt` (TIMESTAMPTZ).
* `updatedAt` (TIMESTAMPTZ).

## Out of Scope & Limitations
* **Valuation & Costing**: No inventory valuation, WAC calculation, food costing, or recipe rollup are implemented.
* **Warehouse / Locations**: Single stock container assumption (no multiple warehouses or locations).
* **Multi-Branch Readiness**: No branch ownership, multi-branch filters, or branch permissions. RFC-0010 remains Proposed.
* **Supabase Integration**: Stores changes to `localStorage` for mocking capabilities.
