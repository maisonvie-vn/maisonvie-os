# TASK-0012 — Improvement Action Foundation

## Objective
The objective of TASK-0012 is to build the **Improvement Action Foundation** inside MVOS Studio. This converts operational lessons and root causes from the Continuous Learning module into simple, assignable, and trackable corrective actions ("Hành động cải tiến") with assignees, priorities, and deadlines.

## Data Structure

### Improvement Action Fields
* `id` (UUID): Unique action identifier.
* `learningEventId` (UUID, Nullable): Foreign key to the associated `learning_events` record.
* `title` (VARCHAR): Summary of the corrective work.
* `description` (TEXT): Detailed description of tasks.
* `ownerName` (VARCHAR): Assigned person.
* `priority` (VARCHAR): Severity priority (`low`, `medium`, `high`, `critical`).
* `status` (VARCHAR): Action status (`open`, `in_progress`, `blocked`, `completed`, `cancelled`).
* `dueDate` (DATE, Nullable): Completion deadline.
* `completedAt` (TIMESTAMP, Nullable): Completion timestamp.
* `resolutionNote` (TEXT, Nullable): Closing review comments when marked resolved.
* `createdAt` / `updatedAt` (TIMESTAMP).

## Features Implemented

### 1. Summary Cards
Shows real statistics calculated from the actions list:
* **Tổng hành động**: Total improvement actions logged.
* **Đang xử lý**: Actions with status `in_progress`.
* **Đang bị chặn**: Actions with status `blocked`.
* **Quá hạn**: Unfinished actions past their due date.
* **Đã hoàn thành**: Resolved actions with status `completed`.

### 2. Action Registration Form
Allows logging new improvement actions with peer learning links. Includes input validations in Vietnamese.

### 3. Detailed View & Action Panel
Displays descriptions, dates, and resolution notes, and allows updating action status (`Chuyển sang đang xử lý`, `Đánh dấu đang bị chặn`, `Đánh dấu đã hoàn thành` with comments, and `Hủy hành động`).

## Out of Scope & Limitations
* **Automatic Staff Evaluation**: No staff scoring or automatic performance evaluations.
* **AI Recommendations**: No automated incident-action routing or generative work descriptions.
* **Supabase Integration**: Currently stores changes to `localStorage` for mocking capabilities. Database triggers and real client queries will be wired in future development stages.
