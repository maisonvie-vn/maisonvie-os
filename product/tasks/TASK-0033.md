# TASK-0033 — WAC Unit Cost Foundation

## Objective
The objective of TASK-0033 is to implement the **WAC Unit Cost Foundation** in MVOS Studio. This enables Maison Vie to calculate the Weighted Average Cost (WAC) unit cost per ingredient based on accepted goods receipt quantities and unit prices, leveraging the Unit Conversion Foundation to normalize quantities to the ingredient costing/default unit.

## Data Structure

### WAC Run Fields
* `id` (UUID): Unique run identifier.
* `runNumber` (VARCHAR): Unique run number (e.g. WAC-20260630-01).
* `runDate` (DATE): Run date.
* `status` (VARCHAR): Status enum (`draft`, `completed`, `completed_with_warnings`, `failed`, `cancelled`).
* `notes` (TEXT, Nullable).
* `createdBy` (VARCHAR, Nullable).
* `startedAt` (TIMESTAMPTZ, Nullable).
* `completedAt` (TIMESTAMPTZ, Nullable).
* `createdAt` (TIMESTAMPTZ).
* `updatedAt` (TIMESTAMPTZ).

### WAC Item Fields
* `id` (UUID): Unique WAC calculation record identifier.
* `wacRunId` (UUID): Reference to parent WAC run.
* `ingredientMasterId` (UUID): Reference to Ingredient Master.
* `costingUnit` (VARCHAR, Nullable).
* `totalAcceptedQuantity` (NUMERIC).
* `totalNormalizedQuantity` (NUMERIC).
* `totalPurchaseCost` (NUMERIC).
* `wacUnitCost` (NUMERIC, Nullable): Calculated WAC unit cost.
* `calculationStatus` (VARCHAR): Status enum (`pending`, `calculated`, `missing_costing_unit`, `missing_conversion`, `missing_price`, `no_receipts`, `incomplete`, `error`).
* `warningMessage` (TEXT, Nullable).
* `sourceLineCount` (INT).
* `createdAt` (TIMESTAMPTZ).
* `updatedAt` (TIMESTAMPTZ).

## Out of Scope & Limitations
* **COGS & Accounting**: No accounting COGS, recipe costing, menu item food cost calculations, gross profit dashboard, or inventory valuation reports.
* **Direct Cost Mutation**: WAC unit costs are strictly read-only and derived from receipt and conversion data.
* **Warehouse / Locations**: Single stock container assumption.
* **Multi-Branch Readiness**: No branch ownership, multi-branch filters, or branch permissions. RFC-0010 remains Proposed.
* **Supabase Integration**: Stores changes to `localStorage` for mocking capabilities.
