# TASK-0009 — Continuous Learning Foundation

## Objective
The objective of TASK-0009 is to build the **Continuous Learning Foundation** inside MVOS Studio. This feature provides a structured learning log ("Nhật ký học tập vận hành") for Maison Vie's management to record operational incidents, root causes, lessons learned, and proposed actions from real-world restaurant operations.

## Data Structure

### Learning Event Fields
* `id` (UUID): Unique identifier.
* `title` (VARCHAR): Summary of the incident/lesson.
* `description` (TEXT): Detailed description of the problem.
* `sourceType` (VARCHAR): Source classification (`reservation`, `email`, `dashboard`, `guest_feedback`, `service`, `kitchen`, `finance`, `inventory`, `manual`).
* `category` (VARCHAR): Problem category (`guest_experience`, `service_quality`, `kitchen_quality`, `reservation_process`, `email_process`, `communication`, `cost_control`, `staffing`, `compliance`, `system`, `other`).
* `severity` (VARCHAR): Severity classification (`low`, `medium`, `high`, `critical`).
* `status` (VARCHAR): Event lifecycle status (`open`, `reviewing`, `action_required`, `resolved`, `archived`).
* `ownerName` (VARCHAR): Personnel assigned to review/resolve.
* `lesson` (TEXT): Lessons learned and insights.
* `proposedAction` (TEXT): Corrective action proposed to prevent recurrence.
* `createdAt` / `updatedAt` (TIMESTAMP).

## Features Implemented

### 1. Summary Cards
Shows real statistics calculated from the learning events:
* **Tổng bài học**: Total number of logged learning events.
* **Cần hành động**: Total events with status `action_required`.
* **Đang xem xét**: Total events with status `reviewing`.
* **Đã xử lý**: Total events with status `resolved`.
* **Nghiêm trọng / Cao**: Total events with severity `high` or `critical`.

### 2. Learning Event Registration Form
Allows users to log a new learning event. Includes frontend input validation in Vietnamese (requires title, description, category, and severity).

### 3. Detailed View & Action Panel
Enables users to click any event to read its details (descriptions, lessons, proposed actions) and execute lifecycle state mutations (`Chuyển sang đang xem xét`, `Đánh dấu cần hành động`, `Đánh dấu đã xử lý`, `Lưu trữ`).

---

## Out of Scope & Limitations
* **AI Recommendations**: No automatic lesson summaries or AI vector search.
* **Incident Auto-trigger**: Events are logged manually or linked by reference IDs. Auto-generation from reservations is out of scope.
* **Supabase Integration**: Currently stores changes to `localStorage:mvos_learnings` for mock capabilities. Future steps will bind database queries to the Supabase client.
